MVP Human Test Checklist

1. Sidebar and routing
- Verify the sidebar loads all visible campaigns correctly.
- Verify the sidebar stays sticky while the main panel scrolls.
- Verify the All Accounts sidebar link opens the correct page.
- Verify deleted campaigns disappear from the sidebar and cannot be reopened by URL.

2. Create campaign
- Verify the create-campaign modal opens and closes normally.
- Verify name, niche, description, and icon submission works.
- Verify duplicate campaign names or slugs are rejected clearly.
- Verify successful creation redirects into the new campaign.
- Verify the new campaign still exists after a full reload.

3. Delete campaign
- Verify the delete button is visible on campaign pages.
- Verify delete confirmation behavior is correct.
- Verify deleting a campaign removes it from the sidebar.
- Verify campaign posts for that campaign are cleaned up.
- Verify accounts previously assigned to that campaign are released to idle in data.

4. Campaign overview
- Verify total accounts, posted content, and queued content counts render correctly.
- Verify clicking the metric cards opens the correct detail pages.
- Verify campaign overview loads without layout or hydration issues.

5. Add account
- Verify the add-account modal opens and closes normally.
- Verify username is required.
- Verify duplicate usernames in the same campaign are rejected.
- Verify the created account is automatically Instagram.
- Verify the created account inherits the campaign niche.
- Verify the account appears in the campaign account list after creation.

6. Add post
- Verify Add Post is disabled when the campaign has no assigned accounts.
- Verify the add-post modal accepts content, publish date, and video.
- Verify successful creation schedules through PostOnce with publish_at.
- Verify one post targets all assigned campaign accounts.
- Verify the post appears in queued content afterward.

7. Queued posts table
- Verify queued rows load correctly on the detail page.
- Verify live PostOnce post detail is shown correctly.
- Verify target usernames and target statuses render correctly.
- Verify inline publish_at editing works from the table.
- Verify row actions render correctly for every queued row.

8. Queued post edit
- Verify the editor modal opens from the queued table.
- Verify caption changes save correctly.
- Verify publish_at changes save correctly.
- Verify video preview/player loads correctly.
- Verify replacing the video creates a new scheduled post and removes the old one.
- Verify the row updates to the new post ID after replacement.

9. Queued post delete
- Verify deleting a queued post removes it from the table.
- Verify the PostOnce scheduled post is deleted remotely.
- Verify the counts on campaign overview/details stay correct after deletion.

10. Bulk publish
- Verify bulk publish is available on non-Kick campaigns.
- Verify bulk publish is not shown on Kick campaign.
- Verify start date and folder path validation works.
- Verify progress UI works during a real run.
- Verify bulk-created posts follow the campaign cadence rules.
- Verify generated queued posts appear in the queued table.

11. Kick scan
- Verify the scan feature only appears on Kick campaign.
- Verify the scan button disables while scanning.
- Verify the progress bar is shown during scan.
- Verify scan errors appear in snackbar.
- Verify successful results render as interactive account cards.
- Verify copy-output and copy-line actions work.
- Verify CSV download works.
- Verify per-account formatting follows: username: comma-separated links.
- Verify each line contains at most 10 links before line wrapping to the next account line.

12. All Accounts page
- Verify only Instagram accounts appear.
- Verify columns render correctly: username, platform, niche, status, assigned campaign, id.
- Verify status behavior is correct:
- active when assigned
- inactive when source account is inactive and not assigned
- idle in data after campaign deletion
- Verify search works across username, niche, status, campaign, and id.
- Verify current page behavior still hides idle rows if that is intended.

13. Timezone handling
- Verify manual publish_at input is interpreted in EST.
- Verify edited publish_at values remain correct in EST.
- Verify actual scheduled time in PostOnce matches the intended EST time.

14. Snackbar and error handling
- Verify errors show in snackbar for:
- create campaign
- delete campaign
- add account
- add post
- edit queued post
- delete queued post
- bulk publish
- kick scan
- Verify success feedback also appears where expected.

15. Persistence and reload
- Reload after creating a campaign and verify it persists.
- Reload after deleting a campaign and verify it stays deleted.
- Reload after creating an account and verify it persists.
- Reload after creating a post and verify it persists.
- Reload after editing a queued post and verify it persists.
- Reload after deleting a queued post and verify it stays removed.

Highest live-risk areas
- PostOnce create, patch, delete, and media replacement behavior with the real token
- Instagram scan reliability under 429 rate limits or anonymous-access blocking
- Bulk publish against real folders, real media files, and long-running jobs
